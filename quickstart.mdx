Get your nexDash dashboard up and running quickly with this step-by-step guide.

## Prerequisites

Before you begin, ensure you have:

- Node.js 18.17.0 or higher
- npm, yarn, or pnpm package manager
- A Vercel Postgres database (or any PostgreSQL database)
- Git installed on your machine

## Get started

<Steps>
  <Step title="Clone the repository">
    Clone the nexDash repository to your local machine:

    <CodeGroup>
    ```bash HTTPS
    git clone https://github.com/Harshit-Khattar/nexDash.git
    cd nexDash
    ```

    ```bash SSH
    git clone git@github.com:Harshit-Khattar/nexDash.git
    cd nexDash
    ```

    ```bash GitHub CLI
    gh repo clone Harshit-Khattar/nexDash
    cd nexDash
    ```
    </CodeGroup>
  </Step>

  <Step title="Install dependencies">
    Install all required dependencies:

    <CodeGroup>
    ```bash npm
    npm install
    ```

    ```bash yarn
    yarn install
    ```

    ```bash pnpm
    pnpm install
    ```
    </CodeGroup>

    This will install:
    - Next.js 14 and React 18
    - TypeScript and type definitions
    - Tailwind CSS for styling
    - Vercel Postgres SDK
    - bcrypt for password hashing
    - Additional UI dependencies
  </Step>

  <Step title="Set up environment variables">
    Create a `.env.local` file in the root directory:

    ```bash
    cp .env.example .env.local
    ```

    Update the file with your database credentials:

    ```bash
    POSTGRES_URL="postgresql://..."
    POSTGRES_PRISMA_URL="postgresql://..."
    POSTGRES_URL_NON_POOLING="postgresql://..."
    POSTGRES_USER="your_user"
    POSTGRES_HOST="your_host"
    POSTGRES_PASSWORD="your_password"
    POSTGRES_DATABASE="your_database"

    AUTH_SECRET="your_secret_here"
    AUTH_URL="http://localhost:3000/api/auth"
    ```

    <Tip>
    Generate a secure `AUTH_SECRET` using: `openssl rand -base64 32`
    </Tip>

    See [environment variables](/development/environment-variables) for detailed configuration.
  </Step>

  <Step title="Seed the database">
    Populate your database with sample data:

    ```bash
    npm run seed
    ```

    This creates and populates four tables:
    - **users** - 1 demo user
    - **customers** - 10 sample customers
    - **invoices** - 15 sample invoices
    - **revenue** - 12 months of revenue data

    <Note>
    The demo credentials are:
    - Email: `user@nextmail.com`
    - Password: `123456`
    </Note>

    See [database setup](/development/database-setup) for more details.
  </Step>

  <Step title="Start the development server">
    Run the development server:

    <CodeGroup>
    ```bash npm
    npm run dev
    ```

    ```bash yarn
    yarn dev
    ```

    ```bash pnpm
    pnpm dev
    ```
    </CodeGroup>

    The application will start at `http://localhost:3000`.
  </Step>

  <Step title="Log in to the dashboard">
    1. Open your browser to `http://localhost:3000`
    2. Click **Log in** on the landing page
    3. Enter the demo credentials:
       - Email: `user@nextmail.com`
       - Password: `123456`
    4. You'll be redirected to the dashboard

    You can now explore:
    - Customer management
    - Invoice tracking
    - Revenue charts
    - Dashboard analytics
  </Step>

  <Step title="Explore the features">
    Navigate through the dashboard:

    </CardGroup>
  </Step>
</Steps>

## Available scripts

| Script | Command | Description |
|--------|---------|-------------|
| Development | `npm run dev` | Start development server on port 3000 |
| Build | `npm run build` | Create production build |
| Start | `npm run start` | Run production server |
| Format | `npm run prettier` | Format code with Prettier |
| Format check | `npm run prettier:check` | Check code formatting |

## Troubleshooting

<AccordionGroup>
  <Accordion title="Database connection failed">
    Ensure your `.env.local` file has correct database credentials. Check that:
    - The database URL is properly formatted
    - Your database server is running
    - Network access is allowed to the database host
  </Accordion>

  <Accordion title="Seed script errors">
    If seeding fails:
    - Verify database connection first
    - Check that tables don't already exist (drop them if needed)
    - Ensure the database user has CREATE TABLE permissions
  </Accordion>

  <Accordion title="Port 3000 already in use">
    Either:
    - Stop the process using port 3000
    - Or use a different port: `PORT=3001 npm run dev`
  </Accordion>

  <Accordion title="Authentication not working">
    Make sure:
    - `AUTH_SECRET` is set in `.env.local`
    - The users table has been seeded
    - You're using the correct demo credentials
  </Accordion>
</AccordionGroup>
