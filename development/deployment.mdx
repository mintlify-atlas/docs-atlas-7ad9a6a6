nexDash is optimized for deployment on Vercel, taking full advantage of Next.js 14 features and Vercel Postgres integration.

## Quick deploy

The fastest way to deploy nexDash is directly through Vercel:

<Steps>
  <Step title="Push to Git">
    Ensure your code is pushed to a Git repository (GitHub, GitLab, or Bitbucket).

    ```bash
    git add .
    git commit -m "Ready for deployment"
    git push origin main
    ```
  </Step>

  <Step title="Import to Vercel">
    1. Go to [vercel.com/new](https://vercel.com/new)
    2. Import your repository
    3. Vercel will automatically detect Next.js configuration
  </Step>

  <Step title="Configure environment variables">
    Add your environment variables in the Vercel dashboard:

    - `POSTGRES_URL`
    - `POSTGRES_PRISMA_URL`
    - `POSTGRES_URL_NON_POOLING`
    - `POSTGRES_USER`
    - `POSTGRES_HOST`
    - `POSTGRES_PASSWORD`
    - `POSTGRES_DATABASE`
    - `AUTH_SECRET`
    - `AUTH_URL`

    See [environment variables](/development/environment-variables) for detailed configuration.
  </Step>

  <Step title="Deploy">
    Click "Deploy" and Vercel will build and deploy your application.
  </Step>
</Steps>

<Note>
  Vercel automatically runs `next build` during deployment. No additional build configuration is required.
</Note>

## Build process

The build process follows these steps:

<Steps>
  <Step title="Install dependencies">
    Vercel automatically detects `package.json` and installs dependencies:

    ```bash
    npm install
    ```
  </Step>

  <Step title="Run Next.js build">
    The build command from `package.json` is executed:

    ```bash
    npm run build
    ```

    This runs `next build`, which:
    - Compiles TypeScript to JavaScript
    - Optimizes and bundles all pages and components
    - Generates static assets
    - Creates server-side rendering (SSR) bundles
  </Step>

  <Step title="Deploy to edge network">
    Vercel deploys your application to its global edge network for optimal performance.
  </Step>
</Steps>

## Deployment configuration

### Node.js version

nexDash requires Node.js 18.17.0 or higher, as specified in `package.json`:

```json
"engines": {
  "node": ">=18.17.0"
}
```

Vercel automatically respects this requirement.

### Build settings

Vercel automatically detects the Next.js framework and uses optimal settings:

| Setting | Value |
|---------|-------|
| Framework Preset | Next.js |
| Build Command | `next build` |
| Output Directory | `.next` |
| Install Command | `npm install` |
| Development Command | `next dev` |

## Environment setup

### Production environment variables

<Warning>
  Never commit `.env.local` or any file containing secrets to your repository.
</Warning>

Set environment variables in the Vercel dashboard under **Settings → Environment Variables**:

<CodeGroup>
```bash Production
# Database (from Vercel Postgres dashboard)
POSTGRES_URL="postgres://..."
POSTGRES_PRISMA_URL="postgres://..."
POSTGRES_URL_NON_POOLING="postgres://..."
POSTGRES_USER="default"
POSTGRES_HOST="...-pooler.postgres.vercel-storage.com"
POSTGRES_PASSWORD="..."
POSTGRES_DATABASE="verceldb"

# Authentication
AUTH_SECRET="your-production-secret"
AUTH_URL="https://your-domain.vercel.app/api/auth"
```

```bash Preview
# Same as production, but with separate AUTH_URL
AUTH_URL="https://your-preview-deployment.vercel.app/api/auth"
```

```bash Development
# Use preview/development database
AUTH_URL="http://localhost:3000/api/auth"
```
</CodeGroup>

### Generating AUTH_SECRET

Generate a secure authentication secret using OpenSSL:

```bash
openssl rand -base64 32
```

This creates a cryptographically secure random string for JWT signing.

## Database setup on Vercel

<Steps>
  <Step title="Create Postgres database">
    1. Go to your Vercel dashboard
    2. Navigate to the **Storage** tab
    3. Click **Create Database**
    4. Select **Postgres**
    5. Choose your region (select closest to your users)
  </Step>

  <Step title="Connect to your project">
    1. Select your nexDash project
    2. Vercel automatically adds all database environment variables
  </Step>

  <Step title="Seed the database">
    After your first deployment, seed the database:

    ```bash
    # Install Vercel CLI
    npm i -g vercel

    # Pull environment variables
    vercel env pull .env.local

    # Run seed script
    node scripts/seed.js
    ```

    See [database setup](/development/database-setup) for more details.
  </Step>
</Steps>

## Available scripts

nexDash includes several npm scripts for development and deployment:

```json
{
  "scripts": {
    "build": "next build",
    "dev": "next dev",
    "start": "next start",
    "prettier": "prettier --write --ignore-unknown .",
    "prettier:check": "prettier --check --ignore-unknown ."
  }
}
```

### Script descriptions

| Script | Command | Description |
|--------|---------|-------------|
| `dev` | `npm run dev` | Starts development server on `localhost:3000` |
| `build` | `npm run build` | Creates production build |
| `start` | `npm run start` | Starts production server (requires build first) |
| `prettier` | `npm run prettier` | Formats all files |
| `prettier:check` | `npm run prettier:check` | Checks if files are formatted |

## Vercel CLI deployment

You can also deploy using the Vercel CLI:

<Steps>
  <Step title="Install Vercel CLI">
    ```bash
    npm i -g vercel
    ```
  </Step>

  <Step title="Login to Vercel">
    ```bash
    vercel login
    ```
  </Step>

  <Step title="Deploy">
    <CodeGroup>
    ```bash Preview Deployment
    vercel
    ```

    ```bash Production Deployment
    vercel --prod
    ```
    </CodeGroup>
  </Step>
</Steps>

## Post-deployment checklist

After deploying, verify these items:

- [ ] Application loads successfully
- [ ] Database connection is working
- [ ] Authentication flow works correctly
- [ ] Environment variables are set correctly
- [ ] Custom domain is configured (if applicable)
- [ ] SSL certificate is active

## Continuous deployment

Vercel automatically deploys your application when you push to your Git repository:

- **Production deployments**: Triggered by pushes to your main branch
- **Preview deployments**: Created for every pull request

<Note>
  Each preview deployment gets a unique URL, perfect for testing changes before merging.
</Note>

## Performance optimization

nexDash is built with Next.js 14 and includes several performance optimizations:

- **Automatic code splitting**: Each page only loads necessary JavaScript
- **Image optimization**: Next.js Image component with automatic WebP conversion
- **Server components**: Default server-side rendering for optimal performance
- **Edge caching**: Static assets cached on Vercel's global CDN

## Monitoring and logs

Access deployment logs and runtime monitoring:

1. Go to your project dashboard on Vercel
2. Click on a deployment to view build logs
3. Use the **Logs** tab for runtime logs
4. Check **Analytics** for performance metrics

## Rollback

If you need to rollback a deployment:

1. Go to your project's **Deployments** page
2. Find a previous successful deployment
3. Click the three-dot menu
4. Select **Promote to Production**

<Warning>
  Rollbacks only affect your application code, not database state. Database migrations may need manual reversal.
</Warning>
