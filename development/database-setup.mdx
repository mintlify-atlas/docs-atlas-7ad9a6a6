nexDash uses Vercel Postgres as its database solution. This guide walks you through setting up your database and populating it with initial data.

## Prerequisites

Before setting up your database, ensure you have:

- A Vercel account with a Postgres database created
- Database connection strings from your Vercel dashboard
- Node.js 18.17.0 or higher installed

<Note>
  You can create a Postgres database from the [Vercel Dashboard](https://vercel.com/dashboard) under the Storage tab.
</Note>

## Database schema

The nexDash database consists of four main tables:

### Users table

Stores user authentication and profile information.

```sql
CREATE TABLE IF NOT EXISTS users (
  id UUID DEFAULT uuid_generate_v4() PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  email TEXT NOT NULL UNIQUE,
  password TEXT NOT NULL
);
```

### Customers table

Stores customer information for invoice management.

```sql
CREATE TABLE IF NOT EXISTS customers (
  id UUID DEFAULT uuid_generate_v4() PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  email VARCHAR(255) NOT NULL,
  image_url VARCHAR(255) NOT NULL
);
```

### Invoices table

Stores invoice data linked to customers.

```sql
CREATE TABLE IF NOT EXISTS invoices (
  id UUID DEFAULT uuid_generate_v4() PRIMARY KEY,
  customer_id UUID NOT NULL,
  amount INT NOT NULL,
  status VARCHAR(255) NOT NULL,
  date DATE NOT NULL
);
```

### Revenue table

Stores monthly revenue data for dashboard analytics.

```sql
CREATE TABLE IF NOT EXISTS revenue (
  month VARCHAR(4) NOT NULL UNIQUE,
  revenue INT NOT NULL
);
```

## Seeding the database

The seed script automatically creates tables and populates them with sample data.

<Steps>
  <Step title="Configure environment variables">
    Ensure your database connection strings are set in `.env.local`. See [environment variables](/development/environment-variables) for details.
  </Step>

  <Step title="Run the seed script">
    Execute the seed script from your project root:

    ```bash
    node scripts/seed.js
    ```

    The script will:
    - Create the `uuid-ossp` extension for UUID generation
    - Create all four database tables
    - Insert sample data from `app/lib/placeholder-data.js`
    - Hash user passwords using bcrypt
  </Step>

  <Step title="Verify the seed">
    You should see output similar to:

    ```bash
    Created "users" table
    Seeded 1 users
    Created "customers" table
    Seeded 10 customers
    Created "invoices" table
    Seeded 15 invoices
    Created "revenue" table
    Seeded 12 revenue
    ```
  </Step>
</Steps>

## Seed script details

The seed script (`scripts/seed.js`) contains four main functions:

### seedUsers()

Creates the users table and inserts user accounts with hashed passwords.

```javascript
async function seedUsers(client) {
  await client.sql`CREATE EXTENSION IF NOT EXISTS "uuid-ossp"`;
  
  const createTable = await client.sql`
    CREATE TABLE IF NOT EXISTS users (
      id UUID DEFAULT uuid_generate_v4() PRIMARY KEY,
      name VARCHAR(255) NOT NULL,
      email TEXT NOT NULL UNIQUE,
      password TEXT NOT NULL
    );
  `;

  const insertedUsers = await Promise.all(
    users.map(async (user) => {
      const hashedPassword = await bcrypt.hash(user.password, 10);
      return client.sql`
        INSERT INTO users (id, name, email, password)
        VALUES (${user.id}, ${user.name}, ${user.email}, ${hashedPassword})
        ON CONFLICT (id) DO NOTHING;
      `;
    }),
  );
}
```

<Warning>
  Passwords are hashed using bcrypt with a salt round of 10 before being stored in the database.
</Warning>

## Sample data

The seed script uses placeholder data from `app/lib/placeholder-data.js`:

- **1 user**: Default login credentials (email: `user@nextmail.com`)
- **10 customers**: Sample customer profiles with names and emails
- **15 invoices**: Mix of paid and pending invoices across different customers
- **12 revenue entries**: Monthly revenue data for the full year

<Note>
  All seed operations use `ON CONFLICT DO NOTHING` to safely re-run the script without duplicating data.
</Note>

## Troubleshooting

### Connection errors

If you encounter connection errors:

1. Verify your `POSTGRES_URL` is correctly set in `.env.local`
2. Check that your Vercel Postgres database is active
3. Ensure your IP address is allowed (Vercel Postgres allows all IPs by default)

### UUID extension errors

If you see errors about `uuid_generate_v4()`:

```bash
ERROR: function uuid_generate_v4() does not exist
```

The seed script should automatically create the extension, but you can manually enable it:

```sql
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
```

### Re-running the seed

The seed script is idempotent thanks to `ON CONFLICT DO NOTHING` clauses. You can safely re-run it multiple times without creating duplicate records.
